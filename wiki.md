📖 Подсистема: Читатель (Управление доступом)

В данном разделе описан реализованный функционал модуля аутентификации и регистрации для пользователей с ролью **«Читатель»**. 

1. Регистрация в системе

Система предоставляет два гибких сценария для регистрации новых читателей:

-- Регистрация с существующим читательским билетом:
  Система позволяет пользователю зарегистрироваться, привязав к профилю уже имеющийся на руках читательский билет. Предусмотрена строгая валидация формата вводимого билета (соответствие маске `LIB-XXXXXX`).
  
-- Регистрация без читательского билета (Выпуск нового):
  Если у пользователя нет билета, система позволяет пройти регистрацию без него. В этом случае запускается асинхронный фоновый процесс (Worker), который автоматически генерирует уникальный номер нового читательского билета и закрепляет его за профилем пользователя в базе данных.

2. Авторизация (Вход в систему)

Для максимального удобства реализована омниканальная система входа. Читатель может авторизоваться, используя единый пароль и любой из трех идентификаторов на выбор:

-- Вход по Email: Классическая авторизация по адресу электронной почты и паролю.
-- Вход по номеру телефона: Авторизация по номеру телефона и паролю. Под капотом реализована автоматическая нормализация вводимых данных (очистка от пробелов, скобок, тире и приведение форматов `8` и `+7` к единому стандарту), что исключает ошибки при входе.
-- Вход по читательскому билету: Уникальная функция системы, позволяющая использовать номер читательского билета (например, `LIB-123456`) в качестве логина вместе с паролем.

3. Технические особенности реализации
-- Единый эндпоинт логина: Маршрутизатор FastAPI автоматически определяет тип введенного логина (email, телефон или билет) с помощью регулярных выражений (Regex) и формирует корректный SQL-запрос к базе данных PostgreSQL.
-- Безопасность: Пароли пользователей хэшируются с использованием алгоритма SHA-256. Идентификаторы пользователей в БД реализованы через `UUID v4`.
