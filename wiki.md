# Electronic Library

## Описание проекта

Electronic Library - это веб-приложение электронной библиотеки для загрузки, модерации, выдачи и чтения электронных книг. Проект реализован как учебная система с несколькими ролями пользователей: читатель, автор, модератор и администратор.

Главная идея проекта: автор загружает книгу, модератор проверяет файл и описание, после одобрения книга появляется в каталоге, а читатель может взять ее на 14 дней, читать на сайте, скачать файл и сохранять закладки.

## Технологический стек

- Backend: Python, FastAPI
- Frontend: Python, NiceGUI
- Database: PostgreSQL
- Infrastructure: Docker, Docker Compose
- Worker: отдельный фоновый процесс для задач, например генерации читательского билета
- Хранение файлов: локальная папка `backend/uploads`, подключенная в контейнер backend как volume

## Архитектура

Проект разделен на несколько сервисов:

- `db` - база данных PostgreSQL.
- `backend` - REST API на FastAPI.
- `worker` - фоновый обработчик задач.
- `frontend` - пользовательский интерфейс на NiceGUI.

Основные адреса после запуска:

- Сайт: `http://localhost:3000`
- Swagger API: `http://localhost:8000/docs`
- Backend API: `http://localhost:8000`

## Запуск проекта из корня проекта


Роли пользователей
Читатель
Читатель может:

зарегистрироваться как новый пользователь;
войти по email, телефону или читательскому билету;
просматривать каталог опубликованных книг;
искать книги по названию, автору и жанру;
брать книгу на чтение на 14 дней;
смотреть текущие взятые книги;
возвращать книгу;
смотреть историю ранее взятых книг;
открывать книгу на сайте через встроенное окно чтения;
скачивать файл книги;
создавать закладки с номером страницы и заметкой;
переходить к сохраненной закладке.
Закладка сохраняет номер страницы, процент прогресса, подпись позиции и заметку. При нажатии на закладку пользователь открывает страницу чтения нужной книги сразу на сохраненном месте.

Автор
Автор может:

зарегистрироваться и войти в систему;
загрузить новую книгу на модерацию;
указать название, описание, жанр, правообладателя и условия доступа;
прикрепить файл книги в формате PDF или EPUB;
смотреть список своих загруженных книг и их статусы;
видеть, прошла книга модерацию или была отклонена.
Книга автора не попадает сразу в общий каталог. Сначала она получает статус pending_review и отображается у модератора в очереди.

Модератор
Модератор может:

открыть панель модерации;
просматривать очередь книг от авторов;
видеть название, автора, файл, жанр и описание книги;
открыть файл книги прямо из панели модерации;
одобрить книгу;
отклонить книгу с комментарием;
добавить книгу напрямую, минуя очередь автора.
Если модератор одобряет книгу, она получает статус published и становится доступной читателям в каталоге. Если отклоняет - статус меняется на rejected, а комментарий сохраняется в системе.

Администратор
Администратор может:

открыть административную панель;
искать пользователей по имени или email;
менять роли пользователей: reader, author, moderator, admin;
управлять доступом к книгам;
смотреть статистику по книгам и популярности;
видеть данные, полезные для контроля работы библиотеки.
Смена роли логируется в таблицу analysis_jobs.
