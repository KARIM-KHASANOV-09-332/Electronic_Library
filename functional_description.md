Роль: Читатель
Требование
-Регистрация по читательскому билету
-Система должна позволять пользователю с ролью «Читатель» зарегистрироваться в системе, используя существующий читательский билет
-Регистрация без читательского билета
-Система должна позволять пользователю с ролью «Читатель» зарегистрироваться в системе, создавая новый читательский билет
-Авторизация по email
-Система должна позволять пользователю с ролью «Читатель» входить в систему по email и паролю
-Авторизация по номеру телефона
-Система должна позволять пользователю с ролью «Читатель» входить в систему по номеру телефона и паролю
-Авторизация по читательскому билету
-Система должна позволять пользователю с ролью «Читатель» входить в систему по читательскому билету и паролю
Список книг
Система должна позволять пользователю с ролью «Читатель» просматривать в личном кабинете список текущих (взятых на чтение) книг
История 
Система должна позволять пользователю с ролью «Читатель» просматривать историю ранее взятых книг
Закладки
Система должна позволять пользователю с ролью «Читатель» оставлять закладки в книгах
Просмотр каталога книг
Система должна позволять пользователю с ролью «Читатель» просматривать подборки книг на главной странице
Поиск
Система должна позволять пользователю с ролью «Читатель» искать книги по названию, автору или жанру
Чтение
Система должна позволять пользователю с ролью «Читатель» читать книгу
Получение книг
Система должна позволять пользователю с ролью «Читатель» добавлять книгу в свой список чтения на 14 дней

Роль: Модератор
Требование
Просмотр публикуемых книг
Система должна позволять пользователю с ролью «Модератор» просматривать книги, добавленные пользователями с ролью «Автор», до их публикации в общем доступе
Добавление книги
Система должна позволять пользователю с ролью «Модератор» добавлять новые книги минуя лист ожидания, загружая их в виде файла (ERUB, PDF)
Одобрение книги
Система должна позволять пользователю с ролью «Модератор» накладывать положительную резолюцию на книгу и публиковать ее в общий доступ 
Отклонение книги
Система должна позволять пользователю с ролью «Модератор» накладывать отрицательную резолюцию на книгу и сопровождать ее комментарием с указанием причины отказа
Просмотр листов ожидания
Система должна позволять пользователю с ролью «Модератор» просматривать листы ожидания на книги авторов

📚 Спринт 2: Конвейер контента (Загрузка и модерация)
Создаем фундамент системы — сами книги и процесс их публикации.

Добавление книги напрямую (Модератор) — Сделаем это первым, чтобы у нас появилась возможность быстро закидывать тестовые книги в БД без ожидания.

Добавление книги в лист ожидания (Автор) — Загрузка файлов (EPUB, PDF) и сохранение их на сервере.

Редактирование описания книг (Автор) — Добавление метаданных, обложек, жанров.

Указание условия доступа (Автор) — Правообладатели и лицензии.

Просмотр листов ожидания и публикуемых книг (Модератор) — Интерфейс таблицы (NiceGUI) для модератора.

Одобрение и Отклонение книги с комментарием (Модератор) — Изменение статуса книги в БД, после которого она попадает в общий каталог.

Просмотр списка опубликованных книг (Автор) — Личный кабинет писателя.

🔎 Спринт 3: Витрина и выдача книг
Теперь в базе есть одобренные книги, и мы можем пускать туда читателей.

Просмотр каталога книг (Читатель) — Вывод красивых карточек книг на главную страницу.

Поиск (Читатель) — Фильтрация БД по названию, автору или жанру.

Получение книг (Читатель) — Главная бизнес-логика: добавление книги в список чтения на 14 дней с фиксацией даты возврата.

Список книг (Читатель) — Отображение текущих взятых книг в личном кабинете.

📖 Спринт 4: Движок чтения и вовлечение
Пользователь взял книгу, теперь он должен иметь возможность ее читать.

Чтение (Читатель) — Самый технически сложный этап: рендеринг EPUB/PDF файлов прямо в браузере через NiceGUI.

Закладки (Читатель) — Сохранение текущей страницы/процента прочитанного в БД.

История (Читатель) — Отображение книг, у которых истек срок 14 дней или которые пользователь вернул сам.

👑 Спринт 5: Глобальное управление и аналитика
Финальная полировка проекта для сдачи.

Изменение доступа к книгам (Администратор) — Возможность экстренно скрыть книгу (например, по жалобе).

Просмотр статистики книг (Администратор) — Вывод графиков (сколько раз взяли, какие жанры популярнее).
